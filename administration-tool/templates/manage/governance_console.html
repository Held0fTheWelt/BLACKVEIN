Runtime readiness policy

`run_validation_seam` remains canonical. ADR-0041 can veto readiness in bounded scopes, but cannot upgrade reject → allow and cannot take commit authority.

Source: runtime_readiness_consumer.py + ADR-0041

Evidence promotion rule

Local traces/tests/projections are local proof only. Capability promotion to staging/live requires explicit evidence policy gates and external proof.

Source: ADR-0039 + capability_matrix_live_claim_gates.md

What should I choose?

Local development: deterministic validators, `safe_local` posture, optional Langfuse, no staging/live claims.

Staging validation: provider credentials + Langfuse + MCP verification + readiness blockers cleared.

Runtime authority pilot: ADR-0041 plan/projection flags with bounded co-authority diagnostics, still local-only until staging evidence exists.

Production/live: only governance-approved flags, no env-only UI mutations, evidence-backed claims.

Source: ai_engineer_suite_service.py presets + ADR-0041 + capability matrix docs
