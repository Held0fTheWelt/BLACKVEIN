### New Files

#### `backend/app/runtime/scene_legality.py` (110 lines)
**Purpose:** Canonical source of truth for scene/ending legality rules.

**Public API:**
```python
class SceneTransitionLegality:
    @staticmethod
    def check_transition_legal(
        from_scene: str,
        to_scene: str,
        module: ContentModule,
        session: SessionState | None = None,
        detected_triggers: list[str] | None = None,
    ) -> SceneLegalityDecision

    @staticmethod
    def check_ending_legal(
        module: ContentModule,
        session: SessionState | None = None,
        detected_triggers: list[str] | None = None,
    ) -> tuple[str | None, SceneLegalityDecision]
```

**Legality Rules Implemented:**

1. **Scene Transitions:**
   - Self-transitions (staying in current scene) always allowed
   - Target scene must exist in module
   - Either find unconditional transition, or all conditions must be met
   - Return clear reason for allow/deny

2. **Ending Conditions:**
   - Check each ending in order
   - Unconditional endings always triggered
   - Conditional endings require all trigger conditions detected
   - Return ending_id + legality decision

**Error Messages (Canonical):**
- "Self-transition in 'X' is always allowed"
- "Scene 'X' not in module"
- "Scene 'X' not reachable from 'Y'"
- "All transitions from 'X' to 'Y' require unmet conditions"
- "Unconditional ending"
- "All conditions {list} detected"
- "Missing conditions: {list}"

## Coherence Implementation

### Single Canonical Source
```
┌─────────────────────────────────────┐
│  SceneTransitionLegality            │
│  (scene_legality.py)                │
│  - check_transition_legal()         │
│  - check_ending_legal()             │
│  - Canonical rules only             │
└─────────────────────────────────────┘
         ▲              ▲
         │              │
    Uses for       Uses for
    validation     derivation
         │              │
    ┌────┴──┐      ┌────┴──┐
    │        │      │       │
  validate_  →  next_       →
  decision() → situation()

  (validators.py)  (next_situation.py)
```

### Key Properties

1. **No Duplication:** Both paths call identical logic; no separate rule interpretations.

2. **Explicit Separation:** Transition/ending conditions checked in one place.

3. **Testable:** SceneTransitionLegality has clear contracts; easy to test both paths against same rules.

4. **Evolvable:** Changes to legality rules propagate to both validator and derivation automatically.

## Drift Risk Reduction

### Before (Drift-Prone)
- validators.py had custom `_validate_scene_transition()`
- next_situation.py had `_check_transition_condition()` and `_check_ending_condition()`
- Three separate implementations → divergence risk
- Mutations to one path didn't affect others
- AI could potentially validate something that the canonical state machine rejects

### After (Drift-Proof)
- Single `SceneTransitionLegality.check_transition_legal()` called by both paths
- Single `SceneTransitionLegality.check_ending_legal()` called by both paths
- Changes to legality rules affect validator AND derivation simultaneously
- AI proposal validation and canonical state progression use identical rules
- **No path for AI to bypass canonical legality**

### Concrete Risk Scenarios Now Prevented

1. **AI proposes unreachable transition:**
   - Validator calls `check_transition_legal()` → REJECTED
   - Derivation calls `check_transition_legal()` → REJECTED
   - ✅ Consistent

2. **AI proposes illegal ending:**
   - Validator calls `check_ending_legal()` → checks legality
   - Derivation calls `check_ending_legal()` → checks legality
   - ✅ Identical logic

3. **Conditional transition without trigger:**
   - Validator path: checks if unconditional path exists
   - Derivation path: checks if conditions are met or unconditional path exists
   - ✅ Both use same `_check_transition_conditions()` helper
   - ✅ No divergence possible

## Implementation Details

### Self-Transitions
```python
# Step 2: Self-transitions are always allowed (staying in current scene)
if from_scene == to_scene:
    return SceneLegalityDecision(
        allowed=True,
        reason=f"Self-transition in '{from_scene}' is always allowed"
    )
```

**Rationale:** A scene can always "transition" to itself (no progression needed).

### Condition Checking
```python
# Conditional transition: all conditions must be detected
if detected_triggers is not None:
    if all(cond in detected_triggers for cond in transition.trigger_conditions):
        return True, f"All conditions {transition.trigger_conditions} detected"
    else:
        missing = [c for c in transition.trigger_conditions if c not in detected_triggers]
        return False, f"Missing conditions: {missing}"

# Conditions defined but no detected_triggers provided: cannot evaluate
return False, "Conditions undefined but detected_triggers not provided"
```

**Rationale:**
- At validation time: `detected_triggers=None` → checks only if transition exists (not conditions)
- At derivation time: `detected_triggers=[...]` → checks both existence and conditions
