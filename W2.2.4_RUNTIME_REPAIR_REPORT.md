## Executive Summary

Successfully integrated canonical scene transition and ending legality rules into the production turn execution flow. The W2.2.4 gate review identified that validators used `SceneTransitionLegality` but the actual executor applied scene changes through simple existence checks, creating operational incoherence.

**Repair implemented Option B:** Wire canonical legality checks directly into `execute_turn()` without integrating `derive_next_situation()`. This ensures validator and executor make identical decisions about scene and ending legality.

## Solution: Option B Implementation

### 1. Replace Simple Existence Check with Canonical Legality (turn_executor.py:468-495)

**Before:**
```python
if mock_decision.proposed_scene_id in module.scene_phases:
    updated_scene_id = mock_decision.proposed_scene_id
```

**After:**
```python
legality_decision = SceneTransitionLegality.check_transition_legal(
    session.current_scene_id,
    mock_decision.proposed_scene_id,
    module,
    session=session,
    detected_triggers=mock_decision.detected_triggers  # Actual triggers from execution
)
if legality_decision.allowed:
    updated_scene_id = mock_decision.proposed_scene_id
    # Log scene_changed event
else:
    # Log scene_transition_blocked event (for debugging)
```

**Key difference:** Execution has actual `detected_triggers` available, unlike validation which only has `detected_triggers=None`. This allows executor to make better decisions than validator while maintaining canonical legality rules.

### 2. Integrate Ending Legality into Execution (turn_executor.py:497-507)

**Added ending handling:**
```python
ending_id, ending_legality = SceneTransitionLegality.check_ending_legal(
    module,
    session=session,
    detected_triggers=mock_decision.detected_triggers
)
if ending_id and ending_legality.allowed:
    updated_ending_id = ending_id
    # Log ending_triggered event
```

**Result:** Unconditional endings now automatically trigger in execution (not validation-only theater).

### 3. Extend TurnExecutionResult (turn_executor.py:109)

**Added field:**
```python
updated_ending_id: str | None = None
```

**Purpose:** Report which ending was triggered (if any) so callers know if the session has ended.

## Canonical Legality Contract (Final)

### Scene Transition Legality

A scene transition from A to B is legal **if and only if:**

1. **Scene B exists** in the module
2. **Self-transitions** (A == B) are always allowed
3. **Either:**
   - A transition from A to B is defined AND has no trigger_conditions (unconditional), OR
   - A transition from A to B is defined AND all trigger_conditions are in detected_triggers

**Enforced at both points:**
- ✅ **Validator** (with detected_triggers=None): rejects conditionals, accepts unconditionals
- ✅ **Executor** (with actual detected_triggers): accepts if conditions met, rejects otherwise

### Ending Legality

An ending is legal **if and only if:**

1. **Either:**
   - No ending conditions defined at all (no endings available), OR
   - An unconditional ending exists (no trigger_conditions), OR
   - A conditional ending has all trigger_conditions detected

**Enforced at execution time:**
- ✅ Validator prevents illegal ending proposals with detected_triggers=None
- ✅ Executor determines actual ending with detected_triggers from execution

## Implementation Notes

### Why Option B Over Option A

**Option A (Integrate derive_next_situation):**
- Would require wiring next_situation() into execute_turn()
- Adds complexity and coupling
- Better for future when scenario derivation is first-class

**Option B (Wire SceneTransitionLegality checks):**
- ✅ Minimal, focused change (40 lines added to execute_turn)
- ✅ Preserves existing structure
- ✅ Keeps next_situation() as future tool
- ✅ Single point of truth in SceneTransitionLegality
- ✅ Clearer separation of concerns

### Detected Triggers Availability

**Validation time (validators.py):**
- Detects proposed_scene_id but NOT actual triggers from AI response
- Calls `check_transition_legal(..., detected_triggers=None)`
- Conservatively rejects conditionals (correct: can't evaluate without evidence)

**Execution time (turn_executor.py):**
- Has mock_decision.detected_triggers from actual execution
- Calls `check_transition_legal(..., detected_triggers=actual_list)`
- Can evaluate conditionals properly (has evidence)

This asymmetry is correct and intentional.
