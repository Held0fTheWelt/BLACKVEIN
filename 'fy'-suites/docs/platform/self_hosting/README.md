# fy-suites self-hosting overhaul

This directory tracks the self-application pass where the fy-suites are used on the fy-suites themselves.

The goal of this pass is to leave behind a stronger internal picture of:

- canonical Candidate E closure artifacts for D-vs-E proof and final bounded closure claims

- contracts
- structure and hotspots
- code-adjacent documentation
- proof and evidence surfaces
- repository-level generated documentation
- internal tracking and self-observability

Generated self-hosting outputs are expected under Documentify technical and status tracks, especially:

- `technical/SELF_HOSTING_TRACKING.md`
- `technical/SELF_HOSTING_CONTRACTS.md`
- `technical/SELF_HOSTING_EVIDENCE.md`
- `status/SELF_HOSTING_HEALTH.md`
