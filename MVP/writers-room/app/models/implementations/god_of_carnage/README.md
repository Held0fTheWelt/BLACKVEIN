# Implementation Example — Der Gott des Gemetzels

This folder is a **scenario implementation layer** built on top of the generic prompt templates.

## Design Rules
- The reusable templates remain fully generic.
- Scenario-specific content lives only in this implementation folder.
- References use stable IDs, not file paths.
- Canon, adaptation, runtime stacks, and scene beats are split into separate files.
- The subconscious layer is separate from scene direction and player-role identity.

## Scenario Shape
This implementation models a chamber-piece social conflict:
- one host couple
- one guest couple
- one apartment as the primary pressure cooker
- repeated attempts to end the meeting that fail
- escalating alliance shifts
- a social breakdown that never fully resolves

## Active Play Variants
This pack includes four layers:
- `scenario_core.md` for the canonical dramatic structure
- `canon_adaptation_map.yaml` for name and adaptation mapping
- `start_stack_quick.md` and `start_stack_director.md` for immediate runtime use
- player-specific subconscious files for inner-pressure play
