Service overview

This Flask application is the business and integration layer: REST API under /api/v1, JWT/session auth, persistence, moderation, game-content authoring APIs, and HTTP bridging to the world-engine play service. It does not host canonical player HTML or admin management pages.

Architecture (this repo)

  • frontend/ — canonical player/public UI (login, dashboard, play shell in the browser).
  • administration-tool/ — canonical admin/management UI (separate origin in production).
  • backend/ (this service) — API, auth, policy, optional legacy path redirects to FRONTEND_URL, and these /backend/* info pages.
  • world-engine/ (play service) — authoritative live run state, lobby, and WebSocket runtime.
