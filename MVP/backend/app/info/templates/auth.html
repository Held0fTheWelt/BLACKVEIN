JWT (API)

Access tokens are short-lived; refresh tokens are long-lived and revocable (blacklist maintenance runs in production). Algorithm and header names are configured in app.config (JWT_* keys).

Session cookies

Flask server-side sessions may still be touched by legacy compatibility routes (e.g. POST /logout clearing session). Cookie flags: HTTPONLY, SAMESITE=Lax, and SESSION_COOKIE_SECURE when PREFER_HTTPS is enabled — see app.config.Config.

CSRF

Flask-WTF CSRFProtect is enabled application-wide; the api_v1 blueprint is exempt so JSON API clients use JWT instead of form tokens.

CORS

CORS_ORIGINS lists allowed browser origins for cross-origin API access. Include both player frontend and administration-tool origins when those apps run on different hosts/ports than the API.

Frontend / backend / admin boundary

  • frontend/ — player experience; uses API + play service URLs from config.
  • backend/ — API + this technical HTML under /backend; legacy paths redirect to FRONTEND_URL when set.
  • administration-tool/ — management UI; proxies or calls admin API routes with its own session/secret configuration.

Security headers

Responses attach CSP, X-Frame-Options: DENY, nosniff, Referrer-Policy, and optional HSTS when HTTPS enforcement is on — see create_app after_request handler.
