Story Runtime Experience — changed files
==========================================

NEW FILES
---------
ai_stack/story_runtime_experience.py
    Canonical settings model (enums, defaults, normalization, validation,
    StoryRuntimeExperiencePolicy with configured/effective/degradation markers,
    resolver and config extractor). Shared by backend + world-engine.

ai_stack/story_runtime_experience_packaging.py
    Experience-aware visible-output bundle packager. Implements real runtime
    differences between recap, dramatic_turn, and live modes (narration_blocks,
    action_pulses, spoken_lines, scene_motion_summary, continuation_state,
    responder_trace, experience_packaging contract metadata).

backend/app/services/story_runtime_experience_service.py
    Backend service: persistence on scope "story_runtime_experience",
    idempotent baseline seeding, audited update, admin truth-surface builder,
    serializer for resolved runtime config embedding.

administration-tool/static/manage_story_runtime_experience.js
    Admin UI JS: loads truth surface, applies configured values to form,
    renders degradation markers + validation warnings + effective values,
    save path PUTs normalized payload.

tests/integration/test_story_runtime_experience.py
    14 unit tests for config model + policy caps + packaging mode differences.

backend/tests/services/test_story_runtime_experience_service.py
    6 backend integration tests for DB-backed seed + update + truth surface +
    resolved-runtime-config propagation.
