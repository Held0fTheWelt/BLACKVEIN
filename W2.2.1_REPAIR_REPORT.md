## What Was Fixed

### Gap Identified (W2.2 Regression Review)
- `validate_action_structure()` was defined in `validators.py` but never called
- Required field validation happened passively through mutation/reference checks, not explicitly
- No canonical enforcement of action structure before state mutation

### Fix Applied
- Wired `validate_action_structure()` into the canonical `execute_turn_with_ai()` execution path
- Called immediately after action type validation (Step 4b.2)
- Validates all required fields per action type before state mutation
- Returns `system_error` with unchanged state if structure is invalid

### How Validation Happens Now
```
1. AI adapter response parsed → ParsedAIDecision
2. For each ProposedDelta in proposed_deltas:
   a. Validate action type (existing W2.2.1 type checking)
   b. Validate action structure (NEW - W2.2.1 structure checking)
   c. If either invalid, add error and continue
3. If any errors, fail with system_error before state mutation
4. If all valid, proceed to decision_from_parsed() and execute_turn()
```

## Distinction Between Validation Stages

The repair maintains clear semantic boundaries:

| Validation Stage | What It Checks | When | Example |
|---|---|---|---|
| **Action Type Validation** | Is the action type in the canonical taxonomy? | Early (Step 4b.1) | "state_update" is valid |
| **Action Structure Validation** | Does the action have required fields? | Early (Step 4b.2) | "target_path" is present and non-empty |

| **Reference Integrity** | Do referenced entities exist? | Deeper (in execute_turn) | "characters.veronique" exists |
| **Scene Legality** | Is the scene transition allowed? | Deeper/Execution | Scene reachable via phase transitions |
