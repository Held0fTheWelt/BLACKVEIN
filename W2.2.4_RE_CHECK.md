**What is actually wired:**
- Test-only scenario: when you explicitly call both paths with proper parameters, they use identical logic ✅
- **Production scenario**: When actual turn executes:
  1. Validator checks with `detected_triggers=None` → rejects conditional transitions
  2. Turn executor applies accepted deltas and simply checks scene exists → **allows scene change regardless of conditions**
  3. No actual derivation check happens

**Strongest evidence:**
- Conditional transition test calls `derive_next_situation()` explicitly with detected_triggers
- But in production, `derive_next_situation()` is never called
- Production turn executor ignores canonical legality rules

**The Drift:**
```
Validator path (production):
  check_transition_legal(..., detected_triggers=None)
  → rejects conditional transitions

Turn executor (production):
  if proposed_scene_id in module.scene_phases:
    updated_scene_id = proposed_scene_id
  → allows ALL scenes regardless of conditions

Derivation path (test-only):
  check_transition_legal(..., detected_triggers=actual_triggers)
  → checks conditions properly
```

**Critical fact:** An AI could pass validator (which rejects it) but the turn executor would still apply it because:
- If validator rejects, the decision fails validation and execution stops (correct)
- But if validator somehow accepted a conditional transition with `detected_triggers=None`, the turn executor would blindly apply it

**What is missing:**
- **Actual integration**: `derive_next_situation()` is not called in production
- **Conditional transition handling**: Turn executor doesn't check conditions, just existence
- **Runtime coherence**: Validator checks legality but execution doesn't

**Relevant files:**
