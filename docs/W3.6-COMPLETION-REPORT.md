## Implementation Notes

### Test Design Decisions

1. **Behavior-Based Assertions** — Tests verify non-500 responses and graceful failure modes, not exact UX mechanics (e.g., "redirects or shows error" instead of "must redirect")

2. **Redirect Handling** — Turn execution errors naturally redirect to session view; tests use `follow_redirects=True` to verify the final rendered page is usable

3. **Helper Reuse** — Leverage existing test helpers `_create_and_setup_session()` and `_get_csrf_token()` for DRY test code

4. **Import Corrections** — Fixed import from `get_runtime_session` to correct `get_session` from session_store

5. **Minimal Assertions** — Each test verifies only what's required for smoke coverage; no spec over-implementation
