# Scope and Goals

## MVP Objective

Deliver one coherent World of Shadows MVP where authored content, governance, runtime execution, player-facing experience, and operator visibility form one consistent system.

The MVP is a controlled product-learning vehicle, not a broad platform rollout. It must validate that users can feel a meaningful difference between a truth-bound dramatic runtime and generic LLM chat.

## Product Scope

- Keep one product identity: a dramatic runtime platform, not disconnected subsystem demos.
- Preserve the God of Carnage slice as the proof-bearing center.
- Keep the broader target runtime model visible as implementation direction.
- Treat `God of Carnage` as runtime-proof module, not market-proof module.

## In-Scope Capability Groups

- Canonical content authoring, review, publish, and activation flow
- Runtime authority and stage continuity from authored source to player-visible turn output
- Player shell clarity (scene state, action effects, continuity)
- Operator/admin diagnostics, auditability, and controlled interventions
- Governed AI support (retrieval/orchestration/control-plane bounded by runtime authority)
- Minimum viable comparative evaluation against a generic LLM baseline

## Out-of-Scope for MVP Closure

- Full realization of all long-range target layers beyond currently validated slices
- Unbounded autonomous behavior outside governance contracts
- Replacing runtime authority with ad hoc fallback shortcuts

## Non-Negotiable MVP Laws

- No authority ambiguity between authored, published, and runtime-active surfaces
- No hidden content mutation bypassing publish and activation controls
- No deletion of source evidence before preservation mapping and verification are complete
- No claim of implementation completeness without validation evidence

## Success Criteria

- One canonical documentation route exists for MVP implementation
- Sequencing and dependencies are explicit and executable
- Acceptance criteria map directly to testable behavior
- Open obligations are explicit and bounded
- H1-H3 are tested with real evaluator evidence:
  - H1 perceived qualitative difference
  - H2 free-input acceptability within shaped drama
  - H3 visible player value from validation/commit
