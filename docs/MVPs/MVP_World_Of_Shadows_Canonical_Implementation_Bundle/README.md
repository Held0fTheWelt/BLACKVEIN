## Integration State (handoff)

- **Byte reconcile:** compared domains are aligned in the MVP snapshot from active; see `migration_report.md` and repository root [`Task.md`](../../../Task.md).
- **Forward integration:** triage `forward_integration_candidates.md` by domain. Rows targeting **`'fy'-suites/`** refer to **repo-local migration and rules tooling** whose **original lives in the active repo** under that directory; the list highlights gaps where `MVP/'fy'-suites/` still has content the active tree is missing (not a separate product codebase). Other rows may be generated reports elsewhere — review before bulk copy.
- **MVP bundle content (governed import):** for importing prepared MVP bundles, normalizing trees under `mvpify/imports/<id>/normalized`, and mirroring MVP docs into `docs/MVPs/imports/<id>`, use **`mvpify`** — see [`'fy'-suites/mvpify/README.md`](../../../'fy'-suites/mvpify/README.md). That path complements the path-level heuristics in `forward_integration_candidates.md`.
- **mvpify CLI (quick reference):** install the **`fy-suites-autark`** workspace once with `pip install -e .` from [`'fy'-suites/`](../../../'fy'-suites/README.md) (console scripts are declared in [`'fy'-suites/pyproject.toml`](../../../'fy'-suites/pyproject.toml)). Then run `mvpify <inspect|plan|ai-pack|full>` with optional `--repo-root`, `--source-root`, `--mvp-zip`, and `--quiet` (subcommands and flags are defined in [`mvpify/tools/hub_cli.py`](../../../'fy'-suites/mvpify/tools/hub_cli.py)). **Without** the installed script, from **`'fy'-suites/`** on `PYTHONPATH`: `python -m mvpify.tools.hub_cli <subcommand> …` with the same flags. Verb overview: [`'fy'-suites/docs/platform/SUITE_COMMAND_REFERENCE.md`](../../../'fy'-suites/docs/platform/SUITE_COMMAND_REFERENCE.md) (**mvpify** section). Optional **`mvpify-adapter`** entry point: [`mvpify/adapter/cli.py`](../../../'fy'-suites/mvpify/adapter/cli.py).
