# MVP_Live_Runtime_Completion — MVP 4 — Observability, Diagnostics, Langfuse, and Narrative Gov

## Mission

Implement behavior-proof diagnostics, traceable decisions, Langfuse real trace evidence or deterministic local trace export, and Narrative Gov operator truth surfaces for the God of Carnage live path.

This MVP consumes MVP 3 LDSS live-path evidence and makes the operator surfaces prove what actually happened instead of displaying placeholder success.

## Scope

In scope:

- DiagnosticsEnvelope with non-placeholder evidence rules.
- TraceableDecision contract for runtime/AI decisions.
- LangfuseRealTraceEvidence or deterministic local trace export generated by real tests.
- NarrativeGovSummary and administration-tool panels for content health, runtime profile health, runtime module health, LDSS health, frontend render contract readiness, and degradation semantics.
- **Deferred from MVP 2**: Object Admission Override admin UI (`/manage/game/object-admission-config`) with audit trail (see "Deferred Admin Surfaces" section below).
- **Deferred from MVP 2**: State Delta Boundary Override admin UI (`/manage/game/state-delta-config`) with breakglass unlock (see "Deferred Admin Surfaces" section below).
- Degraded outcome semantics that distinguish normal, degraded, failed, and mock/static invalid proof.

Out of scope:

- Frontend staged block renderer implementation; MVP 4 only exposes the render contract diagnostics consumed by MVP 5.
- Rewriting LDSS behavior already implemented in MVP 3, except to add trace/diagnostic instrumentation.

## Base Contract

A real `live_dramatic_scene_simulator` runs in the live play path and renders a staged interactive text-adventure / dramatic chat experience.

The player chooses Annette or Alain. The selected character is human-controlled. The other canonical God of Carnage characters are free NPC dramatic actors. NPCs speak, act, pursue their own line, interact with other NPCs, and interact with the environment through canonical, typical, or `similar_allowed` affordances. NPCs may use admitted objects with or against other actors when valid and non-coercive. The Narrator is the player's inner perception/orientation voice, not a dialogue summarizer.

Diagnostics, Narrative Gov, and Langfuse or deterministic trace export prove the live runtime path. `docker-up.py`, `tests/run_tests.py`, GitHub workflows, and TOML/tooling configs remain fully functional. Partial foundation is not acceptable.

### Global Prohibitions

- Do not reintroduce `visitor` as a story actor, runtime participant, prompt role, responder, lobby seat, frontend role, or fallback alias.
- Do not convert `god_of_carnage_solo` into a content module.
- Do not place canonical God of Carnage story truth in a runtime profile or runtime module.
- Do not allow AI output to speak, act, emote, decide, or physically move for the selected human actor.
- Do not accept legacy blob output as canonical final output.
- Do not accept mock-only Langfuse or hand-written trace JSON as final proof.
- Do not accept field presence as behavior.
- Do not close the MVP if `docker-up.py`, `tests/run_tests.py`, GitHub workflows, or TOML/tooling are broken.
- Do not implement later MVPs except for explicit scaffolds and handoff contracts named in this guide.

## Final Target Dependency

MVP 4 proves the final live path. A trace ID of `null`, empty decision arrays with approved status, static trace exports, or Narrative Gov panels disconnected from real session/run/turn evidence fail this MVP.
