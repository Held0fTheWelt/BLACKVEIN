# Source lineage and concept carry-forward map

## Why this document exists

The current canonical spine is much easier to read than the older full MVP family.
But the older family still contains detailed concept truth that must not become invisible.

## Source-lineage map

### v21/v24 core MVP package
Carries:
- constitutional laws,
- completeness rules,
- reference scaffold role,
- implementation workstreams,
- current-state honesty model.

Carry-forward status:
- still truth-bearing source lineage,
- not the primary active reading spine.

### Memory / emotional / consciousness / recovery family
Carries:
- memory domain framework,
- conflict/lineage/threshold/effect/transformation maps,
- emotional and consciousness layer specifics,
- world-state and relationship memory,
- recovery intelligence and failure-as-story.

Carry-forward status:
- enduring target truth,
- active implementation proof only in partial bridges.

### Writers’ Room / QA / evaluation family
Carries:
- review and recommendation ecosystem,
- gold data/evaluation operations,
- authoring analytics and tuning obligations.

Carry-forward status:
- preserved as governed revision/control-plane truth,
- not fully replayed repository closure.

### Reference scaffold family
Carries:
- executable minimum constitutional slice,
- anti-loss proof for core seams,
- implementation/truth separation discipline.

Carry-forward status:
- proof-preserving companion, not live repo authority.

### Examples, maps, and explanatory assets
Carries:
- session walkthroughs,
- maps and diagrams,
- examples that make broader WoS legible.

Carry-forward status:
- explanatory source lineage and onboarding support,
- not all normative on their own, but not expendable ballast either.

## Anti-loss rule

Future MVP consolidations should be required to answer:
for each older concept family,
where did it go?

Valid answers are only:

- preserved directly,
- preserved as enduring target,
- preserved as proof-preserving companion,
- bounded as mirror/support,
- or kept as explicit tension.

“Not mentioned anymore” is not a valid answer.
