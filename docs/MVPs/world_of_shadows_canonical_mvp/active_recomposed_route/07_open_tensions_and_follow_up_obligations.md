# Open tensions and follow-up obligations

## Final judgment stance

The package is strong enough to support one coherent canonical experience definition.
It is not tension-free.
These tensions must remain visible.

## Tension 1 — backend transitional retirement

The backend still contains compatibility and operator-mirror session surfaces that are explicitly not equivalent to live runtime authority.
This is a real managed residue, not solved merely by wording.

## Tension 2 — player-visible transport convergence

The package has stronger shell continuity than before, but it still does not freshly replay every session-facing transport and observation path as one fully collapsed player contract.

## Tension 3 — player support and accessibility proof

Re-entry and observation are well carried.
Help/recap/save-load/accessibility are still more obligation than fully replayed evidence.

## Tension 4 — Level B evaluator independence

The package still does not support an honest claim of evaluator-independent closure at the stronger Level B standard.

## Tension 5 — source-mirror drift

Root GoC contract mirrors and the embedded `repo/` lane remain structurally risky unless they are kept clearly subordinate.

## Tension 6 — broad target families still exceed active replay

Memory depth, multi-session continuity, authoring analytics, and learned-policy lanes remain source-supported target truth, but not fully replayed active implementation reality.

## Follow-up obligations

1. Close or more sharply quarantine backend transitional runtime residue.
2. Replay a stronger end-to-end player transport contract, not only shell-local continuity.
3. Produce evaluator-grade accessibility/help/recap/save-load evidence or keep them explicitly target/obligation status.
4. Resolve mirror-lane discipline for root GoC mirror docs and `repo/`.
5. Keep broader WoS target families mapped forward so later summaries cannot shrink them away.
