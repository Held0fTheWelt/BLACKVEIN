# Residues and runtime-proof tasks

## Why this document exists

The source set strongly rejects closure theater.
The canonical MVP therefore needs one place where unresolved truth stays visible instead of being silently compressed away.

## Bounded residues that remain active

### 1. Broad platform target vs active slice proof
The enduring WoS target includes more than the current GoC slice proves directly.
That is acceptable as long as the distinction stays explicit.

### 2. Backend transitional retirement is not fully closed
The transitional backend session surface is much narrower and better governed than before, but the evidence still supports a bounded residue rather than full retirement closure.

### 3. Level B evaluator-independent closure is still open
The source set supports Level A closure posture for the active slice, but not a stronger evaluator-independent verdict yet.

### 4. Present-container replay is incomplete
The current audit container can directly rerun scaffold, world-engine, and selected ai-stack tests, but not the Flask-backed backend/frontend route tests.
That means some higher-level player-route proof still depends on packaged evidence rather than fresh replay here.

### 5. Player-facing transport convergence is not fully closed
The committed story path, live-room snapshot path, and operator bundle surfaces are now documented more explicitly, but the supplied evidence still does not justify claiming that every player-visible transport seam has been freshly replayed as one collapsed contract in this container.

### 6. Writers' room / retrieval overlap remains governed residue
The overlap is bounded and governed.
It is not honestly erased.

### 7. Broad memory, multi-session, and authoring ambitions remain target truth
The historical memory framework, emotional/consciousness layers, multi-session continuity model, learned-policy shadow lane, and authoring analytics remain canonical target truth without being equally runtime-proven in the current slice.

## Runtime-proof tasks

### Task 1 — Close the backend transitional retirement residue

**Why:** Needed to move from bounded residue toward full retirement closure.

**Done when:**

- ordinary hardened player path no longer depends on backend-local transitional bootstrap,
- evidence matrix, ledger, and tests all agree,
- and closure can be claimed without compatibility caveats doing hidden authority work.

### Task 2 — Provision and replay the Flask-backed backend/frontend proof surface

**Why:** Current direct reruns still stop at missing Flask-backed route-test dependencies.

**Done when:**

- environment recipe is documented,
- targeted backend/frontend tests rerun successfully or fail informatively for real code reasons,
- and the rerun results align with package-contained proof claims.

### Task 3 — Preserve publish-bound session birth as the only hardened ordinary-player path

**Why:** v17-v18 treat this as a core runtime-maturity law.

**Done when:**

- birth uses published artifact identity on the hardened path,
- resume preserves the same provenance bundle,
- and no silent rebinding occurs across artifact revisions.

### Task 4 — Keep Turn 0, commit truth, and player-route purity jointly provable

**Why:** This is a core constitutional seam for the whole product identity.

**Done when:**

- Turn 0 opening commit remains real,
- the ordinary player route stays commit-aligned,
- route-level tests remain executable in a provisioned environment,
- and operator-only payloads stay out of the player route.

### Task 5 — Strengthen shell-loop, re-entry, and transport-discipline proof

**Why:** Player-shell continuity is product value, not just routing hygiene.

**Done when:**

- shell re-entry and transcript progression remain tested,
- latest committed framing remains visible,
- player-visible transport rules are documented and test-backed,
- and live-room or alternate session feeds do not silently outrank the committed story path.

### Task 6 — Continue explicit governance of writers' room / retrieval overlap

**Why:** The overlap is real and useful, but it is also an anti-drift seam.

**Done when:**

- retrieval remains support rather than truth authority,
- authoring workflows stay subordinate to publish and activation discipline,
- and future refactors do not erase the overlap by documentation theater.

### Task 7 — Define and prove any future dual-surface player path before promotion

**Why:** The archive preserves both committed story playback and live-room style surfaces, but they are not automatically the same product path.

**Done when:**

- either one path becomes canonical and the other remains secondary,
- or both paths are explicitly productized with audience classification, convergence rules, and proof,
- and future documentation does not pretend the distinction does not matter.

### Task 8 — Promote multi-session continuity and learned-policy shadow lane only with real implementation and proof

**Why:** These are canonical target truths, but not current closure claims.

**Done when:**

- multi-session persistence scope is implemented and testable,
- learned policy stays in shadow-lane governance until promotion criteria are met,
- and approval thresholds / rollback / audit trail are real rather than aspirational prose.

### Task 9 — Keep GoC acceptance seams alive through future refactors

**Why:** The current slice is only convincing if its character, pressure, continuity, and relay behavior survive.

**Done when:**

- room/object pressure still matters,
- character performance does not flatten,
- interruption/re-entry and relay patterns remain legible,
- and the acceptance matrix still matches real proof-bearing scenarios.


### Task 10 — Prove preview isolation, package history, and rollback as one operational chain

**Why:** The lifecycle is now restored as canonical MVP truth, but the present archive still does not freshly replay the whole package-governance chain end to end.

**Done when:**

- preview package build and preview execution remain isolated from active runtime truth,
- active-vs-preview comparison is real,
- package history and rollback are executable rather than only documented,
- and promotion / rollback evidence can be replayed without relying only on older package reports.

### Task 11 — Replay runtime-health, notifications, and corrective-recovery visibility in a provisioned admin-capable environment

**Why:** Corrective retry, safe fallback, runtime-health signal, and alert posture are clearly canonical, but the present container cannot honestly certify the full admin-side replay path.

**Done when:**

- retry / fallback telemetry is emitted in a replayable environment,
- runtime-health thresholds and notification behavior are visible on real admin surfaces,
- and the corresponding evidence shows how degradation becomes operator-visible without leaking into the ordinary player route.

### Task 12 — Replay resolved-settings governance, secret handling, and cost posture through the real control plane

**Why:** Settings governance is source-supported and product-relevant, but still not a fresh full-surface closure claim here.

**Done when:**

- bootstrap/trust-anchor, runtime mode, provider/model route selection, and resolved runtime config are replayed through the actual governance path,
- secrets remain write-only and backend-controlled throughout the replay,
- cost / budget / alert behavior is observable on the control plane,
- and the final evidence distinguishes control-plane governance from story-truth authority.
