# World of Shadows canonical MVP

This directory remains the **detailed canonical companion family** for the audited World of Shadows MVP source set.

The **active first-pass route** now starts in [`active_recomposed_route/README.md`](active_recomposed_route/README.md), which promotes the 2026-04-21 recomposition into the actual repository reading path.
This parent directory still matters: it preserves the broader split family, claim/proof maps, GoC acceptance matrices, lifecycle detail, and bounded control-plane material without forcing readers to reconstruct the package from audit bundles.

It is subject-based, preservation-first, and proof-aware:

- it keeps the broader World of Shadows platform identity,
- it keeps God of Carnage as the current proof-bearing vertical slice,
- it keeps unresolved seams visible instead of smoothing them away,
- and it distinguishes canon, support, evidence, and runtime residue so the package stops competing with itself.

## Status terms used in this set

- **Enduring target**: source-supported system truth that remains part of World of Shadows even if the active repository slice does not prove it equally today.
- **Active slice**: the currently strongest, proof-bearing MVP focus in the supplied repository and validation corpus.
- **Implemented now**: supported by repository reality and/or direct executable proof.
- **Package-proven now**: supported by load-bearing validation reports in the supplied archive.
- **Bounded residue**: a known unresolved gap, contradiction, or closure blocker that must stay visible.

## What this repair clarifies

Relative to the earlier consolidation and re-audit passes, this repaired route makes six package-level clarifications explicit:

1. **World of Shadows** is the enduring product identity, while **God of Carnage** remains the current proof-bearing vertical slice.
2. The active recomposed route is the primary first-pass spine; this split canonical family remains the detailed companion authority that supplies deeper binding coverage when the active route points here.
3. Accessibility, recap/help, save/load/re-entry, and operator-safe player support are part of the MVP seam even where proof remains uneven.
4. Emotional, social, and consciousness layers stay canonical because they connect to actual active slice readouts and tests, not because they are vague future flavor.
5. The authoring support tree, audit bundles, runtime payloads, and cached observations are classified as subordinate to canonical authored source, publish identity, and committed runtime truth.
6. The current evidence route is now singular: reviewers should enter via [`../../audit/README.md`](../../audit/README.md), then read the 2026-04-21 repair-implementation bundle and the post-repair re-audit before falling back to older 2026-04-20 repair/consolidation materials as historical support.
