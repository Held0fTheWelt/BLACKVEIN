# Enduring target runtime layers and world model

## Why this document had to be restored

The earlier complete MVPs contained a much broader runtime target than the later slice-centered document layer.
That broader target was still materially relevant, but in the previous canonical reduction it was compressed into a few summary sentences.
That compression risked losing actual design truth.

This document restores the enduring target in a way that stays explicit about implementation status.

## The five-layer runtime target remains canonical intent

The historical complete MVPs describe World of Shadows as a layered runtime rather than a single response generator.
The layers remain canonical target responsibilities:

### Layer 1 — Data and canon
Owns:

- canonical state,
- commit records,
- reveal bundles,
- routing snapshots,
- audit trail,
- checkpoints,
- and governed memory stores.

### Layer 2 — Semantic runtime
Owns:

- intent interpretation,
- workflow orchestration,
- semantic planning and validation support,
- tool selection under governance,
- retrieval consult,
- and post-commit continuity write discipline.

### Layer 3 — Narrative intelligence
Owns:

- beat and pacing logic,
- foreshadowing and callbacks,
- consequence cascades,
- dramatic pressure,
- tonal coherence,
- and scene-level shaping.

### Layer 4 — Emotional and social intelligence
Owns:

- engagement,
- confusion,
- overload,
- support need,
- relationship effects,
- trust and emotional response shaping,
- and player-facing support modulation.

### Layer 5 — Consciousness / meta-strategy
Owns:

- story-health and pacing-health supervision,
- callback debt,
- degraded-mode supervision,
- risk and drift guard,
- recovery intelligence,
- and cross-layer strategy choice.

The active repository slice does not prove all of these equally, but the target model itself remains canonical.

## Memory remains a governed architecture, not a flat context trick

The enduring WoS position on memory is broader than the active slice minimum.
Memory is intended to govern:

- truth and belief,
- relationship continuity,
- social consequence,
- institutional precedent,
- cultural identity,
- rumor and legend,
- trauma and suppression,
- sacred / ontological continuity,
- counter-memory and research,
- and reality-bearing memory only under explicit thresholds and governance.

The historical memory framework also preserved named maps and structures for:

- domains,
- carriers,
- effect surfaces,
- thresholds,
- conflict records,
- lineage,
- transformation,
- and governance.

Those are enduring target structures, not all live runtime claims.

## World-state, relationship, and counterfactual memory

The v13 line added an important clarification that still belongs in the canonical MVP:

- world-state memory,
- temporal relationship memory,
- consequence cascades,
- counterfactual memory,
- and "what was true before turn X but not after" style knowledge.

This layer matters for replay, QA, balancing, and future planning even where the active slice currently proves a narrower continuity lane.

Its preserved target responsibilities include:

- frustration,

It owns:

## Active-slice bridges that already make part of this visible

The broader emotional, social, and consciousness stack is not equally proven end to end, but it is also not disconnected from the active slice.
The current package already exposes partial bridges that matter:

- the active player shell and observation route surface live social readouts such as `social_weather_now`, `live_surface_now`, `carryover_now`, `social_geometry_now`, and `response_pressure_now`,
- targeted GoC tests exercise social-state and semantic-interpretation surfaces directly,
- and runtime-facing shell framing carries bounded "what changed" and "who answers now" style readouts instead of leaving all social logic hidden.

That means the correct reading is:

- **social-state legibility** is already partly active-slice reality,
- **emotional and consciousness supervision** remain broader canonical target layers,
- and support-pressure, recap preference, hint preference, cognitive-load governance, and emotional validation outcome are still under-evidenced enough that they must remain explicit proof burden rather than implied closure.

## NPC autonomy and world simulation

The earlier complete MVPs also preserved a more active world model than the current slice alone shows.
Important NPCs are not supposed to be purely reactive.
The intended target includes:

- full simulation for major NPCs,
- periodic simulation for supporting NPCs,
- event-driven simulation for background actors,
- off-screen events and belief updates,
- and world-pressure updates feeding later narrative planning.

This is still target-state truth, even though the current proof-bearing slice is more room-bound.

## Recovery intelligence and failure-as-story

The v13 line introduced another load-bearing idea that should not disappear:

- not every failure should become a hard block,
- some failures should become new story state,
- the system should preserve recovery playbooks,
- incident-pattern memory,
- failure-as-story continuation possibilities,
- and operator-visible root-cause classification.

This remains part of the enduring World of Shadows identity.

## Multi-session continuity remains canonical target truth

The broader MVP family repeatedly treated continuity as more than “remember the last few turns”.
Across sessions, the system is supposed to define:

- what persists,
- what is abstracted,
- what remains historical only,
- what is intentionally forgotten,
- and what stays module-scoped instead of becoming global runtime truth.

That design matters because WoS is intended to preserve consequence and continuity without turning every past trace into permanently active canon.
The active slice does not yet prove this as a live cross-session runtime feature, but the target truth itself is load-bearing and must stay visible.

## Learned-policy shadow lane remains canonical discipline

The preserved v21 line also carried an important governance rule that later canonical prose still underrepresented:

- learned retrieval heuristics,
- learned improvement policies,
- and similar adaptive runtime aids

must first run in a **shadow lane** rather than on the authoritative player path.

That shadow-lane discipline includes:

- offline comparison,
- rollback path,
- approval thresholds,
- audit trail,
- and feature gating before promotion.

This is not a live-proof claim for the current slice.
It is a canonical discipline for how future adaptive logic may enter the system without violating truth-boundary rules or release honesty.

## Honest status reading

The correct reading is:

- these runtime layers and world-model ambitions remain canonical target truth,
- some of their foundations or analogues exist in the current repo,
- but the active proof-bearing slice does **not** justify presenting the whole layered target as equally implemented now.
