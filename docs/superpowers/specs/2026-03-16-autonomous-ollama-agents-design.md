# Autonomous Ollama Agents for Task.md Completion

**Date:** 2026-03-16
**Status:** Design Approved
**Scope:** REQ D, E, F (API consistency, test coverage, documentation)

---

## 1. Overview

Enable Ollama agents to autonomously complete Task.md requirements D, E, F by:
- Generating code/docs that execute helper functions
- Running full test-driven cycle: baseline → generate → test → verify → commit
- Writing files to `.ollama/` directory
- Promoting validated artifacts to main repository

---

## 2. Goals

✓ Ollama agents write files autonomously (REQ D analysis, REQ E tests, REQ F docs)
✓ Full TDD cycle: run existing tests → generate → test → verify coverage → commit
✓ Both extend existing tests AND create new comprehensive test files
✓ All artifacts generated in `.ollama/` initially, promoted to main repo after validation
✓ Zero manual work - agents handle everything end-to-end

---

## 3. Architecture

### Component Structure

```
.ollama/
├── helper_functions.py              # Functions agents call
├── req_d_e_f_autonomous_agents.py   # Main orchestrator
├── test_suggestion_coverage_complete.py (GENERATED by Agent E)
├── docs_SUGGESTED_DISCUSSIONS.md    (GENERATED by Agent F)
└── CHANGELOG_entry_v0.0.35.md       (GENERATED by Agent F)
```

### Helper Functions API

Agents import and call:
- `write_file(path, content) → bool`
- `run_pytest(test_file, cov_threshold=85) → (bool, float)`
- `verify_coverage(before, after) → bool`
- `git_commit(files, message) → bool`
- `run_existing_tests() → float`

Each function returns success/failure with logging.

---

## 4. Execution Flow

**Per-Agent Flow:**

1. PRE-GENERATION: Run existing tests, capture baseline coverage
2. AGENT CALL: Ollama generates Python code with helper imports
3. CODE EXECUTION: exec(generated_code) in sandbox
4. VERIFICATION: Coverage increased? Files written? Analysis complete?
5. PROMOTION: Copy `.ollama/*` → main repo if validation passes
6. COMMIT: Git commit with coverage metrics

---

## 5. Success Criteria

**REQ D:** Analysis file written, covers consistency + breaking changes
**REQ E:** Tests written, pass, coverage ≥85%, improved from baseline
**REQ F:** Docs + changelog written, all sections present

**Overall:** Zero manual file writes, all committed, production-ready

---

## 6. Safety

- Generated code runs in isolated namespace with timeout
- All writes to `.ollama/` directory
- Promotion to main repo only after validation
- Git commits reviewable before push
