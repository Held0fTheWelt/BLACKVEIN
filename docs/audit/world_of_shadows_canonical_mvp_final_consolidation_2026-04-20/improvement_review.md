# Improvement review

## Decision rule

An improvement is **integrated now** only when it is strongly source-supported and clearly closes an omission or compression gap without speculative redesign.

It is **deferred** when it is justified but still needs stronger proof, environment replay, or more implementation evidence.

It is **rejected** when it conflicts with the one-truth model or broadens scope beyond the audited MVP.
