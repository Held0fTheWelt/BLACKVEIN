# MCP A1.2 Read-Only Tool Set v0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement a read-only operator/dev tooling MCP tool set (A1.2) with real handlers for filesystem and backend health/session operations, plus honest NOT_IMPLEMENTED errors for 501-blocked endpoints.

**Architecture:** Configuration layer (env vars + safe defaults) → HTTP client (requests wrapper with retry) → Filesystem utilities (repo detection + search) → Tool handlers (real logic) → Registry (10 tools: 6 functional + 4 blocked). All handlers are deterministic and safe. Blocked tools fail immediately with NOT_IMPLEMENTED.

**Tech Stack:** Python 3.9+, `requests` (HTTP), `pathlib` (filesystem), pytest with `tmp_path` fixture, json/jsonschema.

---

## File Structure Overview

**Create:**
- `tools/mcp_server/config.py` — Configuration with env vars and defaults
- `tools/mcp_server/backend_client.py` — HTTP client wrapper (5s timeout, 1 retry)
- `tools/mcp_server/fs_tools.py` — Filesystem utilities (repo detection, text search)
- `tools/mcp_server/tests/test_config.py` — Config unit tests
- `tools/mcp_server/tests/test_backend_client.py` — HTTP client tests (mocked requests)
- `tools/mcp_server/tests/test_fs_tools.py` — Filesystem tests (tmp_path)
- `tools/mcp_server/tests/test_tools_handlers.py` — Tool handler tests

**Modify:**
- `tools/mcp_server/tools_registry.py` — Add 10 tool definitions + real handlers
- `tools/mcp_server/README.md` — Document tools, env setup, example calls
