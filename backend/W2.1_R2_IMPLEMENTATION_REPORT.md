## How AI Execution Is Now Wired

### Before W2.1-R2

```
dispatch_turn(execution_mode="ai")
  ↓
execute_turn_with_ai() [called, but no proof it works through dispatcher]
```

### After W2.1-R2

```
dispatch_turn(execution_mode="ai", ai_adapter=adapter)
  ↓
execute_turn_with_ai(session, current_turn, adapter, module)
  ├─ build_adapter_request(session, module) → AdapterRequest
  │
  ├─ adapter.generate(request) → AdapterResponse
  │
  ├─ process_adapter_response(response) → ParseResult
  │  ├─ parse_adapter_response() [parse dict → StructuredAIStoryOutput]
  │  ├─ normalize_structured_output() [normalize to ParsedAIDecision]
  │  └─ prevalidate_decision() [catch obvious errors]
  │
  ├─ decision_from_parsed(parsed) → MockDecision [bridge to runtime]
  │
  ├─ execute_turn(session, turn, decision, module) [RUNTIME VALIDATION]
  │  ├─ validate_decision(decision, module) [engine-controlled validation]
  │  ├─ construct_deltas() [create StateDelta objects]
  │  ├─ apply_deltas() [update canonical_state]
  │  └─ finalize() [create TurnExecutionResult]
  │
  ├─ _create_decision_log() [capture audit log]
  │
  └─ return TurnExecutionResult
```

**Critical flow:** The dispatcher calls `execute_turn_with_ai()`, which delegates to `execute_turn()` for validation. **Runtime validation is not bypassed.** All deltas are validated through the engine's authority before being committed.

## Pipeline Execution Flow

### Step 1: Request Building
```python
request = build_adapter_request(session, module)
# Maps session context to AdapterRequest
```

### Step 2: AI Generation
```python
response = adapter.generate(request)
# Returns AdapterResponse with raw_output + structured_payload
```

### Step 3: Parse & Normalize
```python
parse_result = process_adapter_response(response)
# Validates structure, normalizes to ParsedAIDecision
# Pre-validates for obvious errors (blank fields, duplicates, etc.)
```

### Step 4: Bridge to Runtime
```python
mock_decision = decision_from_parsed(parse_result.decision)
# Converts ParsedAIDecision → MockDecision
# Maps target_path → target (seam between AI and runtime)
```

### Step 5: Runtime Validation & Execution
```python
turn_result = await execute_turn(session, current_turn, decision, module)
# CRITICAL: This is where runtime authority is exercised
# - validate_decision() checks all deltas against module rules
# - construct_deltas() creates StateDelta objects
# - apply_deltas() updates canonical_state
# - finalize() creates TurnExecutionResult
```

### Step 6: Decision Logging
```python
decision_log = _create_decision_log(session, turn, parsed, response, turn_result)
# Captures: raw output, parsed decision, validation outcome, deltas
session.metadata["ai_decision_logs"].append(decision_log)
```

### Step 7: Return Result
```python
return turn_result
# TurnExecutionResult with: status, deltas, state, events
```

## Runtime Authority Preservation

**Key guarantees:**
1. **Validation** — All AI proposals validated by `validate_decision()` before commitment
2. **Delta creation** — Only valid deltas become StateDelta objects
3. **State application** — Only validated deltas update canonical_state
4. **Atomicity** — Either all deltas commit or none (state safety)
5. **Logging** — All decisions logged with acceptance/rejection status

**What this means:**
- AI cannot bypass the runtime engine
- Invalid proposals are caught before state corruption
- All changes are auditable (logged and validated)
- Mock and AI paths have identical safety properties
