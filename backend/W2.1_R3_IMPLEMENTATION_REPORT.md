### Created

#### 1. `backend/app/runtime/adapter_registry.py`
**Responsibility:** Lightweight adapter registration and lookup mechanism.

**Functions:**
- `register_adapter(name, adapter)` — Register adapter by canonical name
- `get_adapter(name)` — Look up adapter by name (case-insensitive)
- `adapter_registered(name)` — Check if adapter is registered
- `clear_registry()` — Clear all registrations (test cleanup)

**Design:**
- Simple global dict-based registry (lightweight, no complexity)
- Case-insensitive lookups (user-friendly)
- Clear error messages when adapter not found

**Why lightweight:** No dependency injection framework, no plugin discovery, no autodiscovery. Just explicit registration.

from app.runtime.adapter_registry import register_adapter

## Adapter Registration and Lookup

### How It Works

1. **Registration:** `register_adapter(name, adapter)` stores adapter in global registry
2. **Lookup:** `get_adapter(name)` retrieves adapter by name (case-insensitive)
3. **Dispatcher Resolution:** When `execution_mode="ai"`, dispatcher looks up `session.adapter_name`
4. **Error Handling:** Clear, specific error if adapter not found

### Example Flow

```
Session created with:
- execution_mode = "ai"
- adapter_name = "test_adapter"

↓

Adapters registered:
register_adapter("test_adapter", DeterministicTestAdapter())

↓

dispatch_turn(session, ...)

↓

Dispatcher checks execution_mode="ai"
Looks up session.adapter_name="test_adapter"
Finds adapter in registry
Executes AI path with resolved adapter
```
